# Gate 1 System
## Administrator Training Manual
### Version 1.0

---

## 1. Purpose

This manual is written ONLY for Admin users.

Admins govern structure and oversight, not day-to-day operations.

---

## 2. Your Role as Admin

You are responsible for:
- Creating events and groups
- Assigning roles to users
- Monitoring system health
- Reviewing escalations
- Reviewing audit logs
- Managing cameras and SD cards

You are NOT responsible for:
- Renaming files
- Editing videos
- Micromanaging editors

---

## 3. Admin Dashboard

### Overview Section
- Total media indexed
- Open issues count
- Pending backups
- Online agents
- Active sessions

### Groups Health
- Issues by group
- Problem patterns
- Leader performance

### Recent Issues
- Critical issues requiring attention
- Escalated items

---

## 4. Event Management

### Creating an Event
1. Go to Events page
2. Click "New Event"
3. Enter name, dates, location
4. Save as Draft
5. Activate when ready

### Event Lifecycle
```
Draft → Active → Completed → Archived
```

### Event Statistics
- Total media
- Before/After videos
- Issues count
- Backup coverage
- Healing cases verified

---

## 5. Group Management

### Creating a Group
1. Go to Groups page
2. Click "New Group"
3. Select event
4. Enter group name
5. Assign leader (optional)
6. Save

### Managing Members
- Add members to groups
- Remove members when needed
- Reassign leaders

### Group Monitoring
- View open issues per group
- Track member activity
- Identify struggling groups

---

## 6. Role Assignment

### Available Roles
| Role | Capabilities |
|------|-------------|
| Admin | Full system access |
| Editor | Copy, rename, report, backup |
| Group Leader | Monitor group, coordinate |
| QA | Review issues, confirm fixes |
| Backup | Verify backups, disk rotation |

### Assigning Roles
1. Go to user management
2. Select user
3. Add/remove roles
4. Save changes

**Role changes are audited.**

---

## 7. Global Search

### Search Capabilities
- Search by name
- Search by condition
- Search by region
- Filter by event
- Filter by status
- Filter by date range

### Search Ethics
- Use responsibly
- Only for legitimate purposes
- All searches are logged

---

## 8. Audit Logs

### What's Logged
- User logins/logouts
- Role changes
- Media access
- Issue actions
- Backup operations
- Search queries
- System changes

### Reviewing Logs
1. Go to Audit section
2. Filter by user/action/date
3. Export if needed

**Logs are immutable.**

---

## 9. Escalation Handling

### When Issues Escalate to You
1. Review the issue details
2. Understand the context
3. Contact relevant parties
4. Make a decision
5. Document resolution

### Escalation Reasons
- Repeated unresolved issues
- Technical failures
- Policy violations
- Resource constraints

---

## 10. System Health Monitoring

### Key Metrics
- Online agents count
- Active sessions
- Backup coverage percentage
- Issue resolution rate
- Camera health scores

### Warning Signs
- Many offline agents
- Low backup coverage
- High issue count
- Repeated escalations

---

## 11. Admin Best Practices

- Use global search responsibly
- Look for patterns, not individuals
- Trust the escalation chain
- Avoid micromanagement
- Document decisions
- Review audits regularly

---

## 12. What Admins Must NOT Do

- ❌ Rename files directly
- ❌ Edit videos
- ❌ Bypass workflows
- ❌ Disable audit logs
- ❌ Share credentials
- ❌ Ignore escalations

---

## 13. Admin Conduct

- Be fair and consistent
- Be structured in decisions
- Be calm under pressure
- Protect system integrity
- Support the teams

---
